# 辅助面板文件管理复制/剪切/粘贴设计

## 背景

当前辅助面板的文件树支持查看、创建、重命名、删除、回滚、复制路径等操作，但还不能在同一工作区文件树内直接复制、剪切和粘贴文件或文件夹。

本需求仅作用于辅助面板里的当前工作区文件树，不扩展到其它目录选择器或全局文件浏览器。

## 目标

- 支持文件和文件夹的复制、剪切、粘贴。
- 粘贴目标优先使用右键所在位置；如果没有右键目标，则使用当前选中项。
- 冲突时先做后端预检；用户可选择“全部覆盖”“全部粘贴成副本”，也可进入逐项处理。
- 将“复制相对路径”“复制绝对路径”与“复制 / 剪切 / 粘贴”统一放入同一个一级菜单下。
- 仅修改辅助面板当前工作区文件树的交互，不影响其它文件管理入口。

## 范围

### 包含

- 文件右键菜单增加复制、剪切、粘贴。
- 文件夹右键菜单增加复制、剪切、粘贴。
- 根目录或文件夹作为目标时支持粘贴。
- 复制相对路径、复制绝对路径与文件操作统一归入一个一级菜单。
- 后端提供统一的粘贴能力，负责文件/文件夹复制与移动。
- 冲突时弹出覆盖/副本选择，并在存在多个冲突项时支持逐项处理。
- 操作完成后刷新文件树。

### 不包含

- 系统级文件管理器集成。
- 目录选择器、上传器、下载器等其它组件中的复制/剪切/粘贴。
- 跨工作区或跨根目录的拖拽式搬运。
- 批量多选文件的同时复制/剪切/粘贴。

## 交互设计

### 右键菜单结构

在辅助面板文件树中，原有“复制文件路径”相关入口改造为一个统一一级菜单，例如：

```text
复制 / 粘贴
  ├─ 复制
  ├─ 剪切
  ├─ 粘贴
  ├─ 复制相对路径
  └─ 复制绝对路径
```

规则：

- 对文件和文件夹都显示该一级菜单。
- `复制`、`剪切` 始终可用。
- `复制相对路径`、`复制绝对路径` 始终可用。
- `粘贴` 仅在存在文件树剪贴板内容时可用。
- 文件右键时，`粘贴` 默认粘贴到该文件所在目录。
- 文件夹或根目录右键时，`粘贴` 粘贴到该目录。

### 冲突预检与提示

粘贴前前端先调用后端冲突预检。后端基于真实文件系统返回冲突列表，不让前端自己递归猜测。

冲突列表至少包含：

- `path`：冲突项相对源条目的路径。
- `sourcePath`：源工作区相对路径。
- `targetPath`：目标工作区相对路径。
- `kind`：`file` 或 `dir`。

弹窗分两层：

1. 摘要层：显示源路径、目标路径、冲突数量，并提供：
   - 全部覆盖
   - 全部粘贴成副本
   - 逐项处理
   - 取消
2. 逐项层：列出每个冲突项，用户可为单项选择：
   - 覆盖
   - 粘贴成副本

选择“全部覆盖”或“全部粘贴成副本”时，前端用统一策略调用粘贴。选择“逐项处理”时，前端把每个冲突项的策略传给后端。

覆盖按钮使用危险操作样式，并必须显示实际将被覆盖的目标路径，避免用户误以为覆盖的是右键目标或工作区根目录。

## 前端架构

前端在辅助面板文件树组件中维护一个局部剪贴板状态，不写入全局共享状态。

建议字段：

- `mode`: `copy` 或 `cut`
- `sourcePath`: 源路径
- `sourceKind`: `file` 或 `dir`
- `sourceName`: 源名称，用于生成副本名和显示提示
- `conflicts`: 最近一次预检返回的冲突列表
- `conflictResolutions`: 逐项处理时每个冲突项的覆盖/副本策略

前端职责：

- 记录用户最后一次“复制”或“剪切”的对象。
- 计算当前右键目标是否可粘贴。
- 调用后端执行粘贴。
- 粘贴前调用后端冲突预检。
- 处理冲突弹窗、全部策略和逐项策略。
- 操作成功后刷新当前文件树。
- 剪切成功后清空剪贴板；复制成功后保留剪贴板，方便连续粘贴。

## 后端架构

后端增加一个统一的文件树粘贴命令，负责真实文件系统操作。

建议命令语义：

- 从 `sourcePath` 向 `targetDirPath` 执行复制或移动。
- 处理文件和文件夹两种类型。
- 检查源路径、目标路径是否都位于当前 `rootPath` 内。
- 检查目标目录合法性。
- 检查是否会把目录粘贴到自身或其子目录中。
- 冲突时按照前端传入的全局策略或逐项策略执行覆盖或副本。
- 提供冲突预检命令，返回后端真实文件系统视角下的冲突列表。

后端的职责必须是原子地处理文件系统变更，不让前端用多次读写拼出“复制文件夹”流程。

## 数据流

### 复制 / 剪切

1. 用户在文件树中右键文件或文件夹。
2. 选择“复制”或“剪切”。
3. 前端把该项信息保存到局部剪贴板状态。
4. 菜单继续保持可用，等待用户粘贴。

### 粘贴

1. 用户在目标文件夹或根目录右键并选择“粘贴”。
2. 前端确定目标目录。
3. 如果没有剪贴板内容，则不执行操作。
4. 前端调用后端粘贴命令。
5. 前端先调用后端冲突预检。
6. 若没有冲突，直接调用后端粘贴命令。
7. 若存在冲突，前端弹出冲突对话框。
8. 用户选择全部覆盖、全部副本或逐项策略后，前端按选择调用后端粘贴命令。
9. 成功后刷新文件树。
10. 若是剪切，清空剪贴板；若是复制，保留剪贴板。

## 错误处理

### 同名冲突

后端预检返回可识别的冲突列表。前端不直接猜测覆盖结果，而是必须让用户选择。若粘贴执行期间又发生新的同名冲突，后端仍返回可识别错误，前端重新提示。

### 非法路径

如果源路径、目标路径或目标父目录不在当前工作区 root 下，直接拒绝并提示错误。

### 危险移动

如果目标目录是源目录本身或其子目录，拒绝操作，避免目录递归嵌套。

### 权限或磁盘错误

例如只读目录、文件被占用、路径不存在等，统一展示后端返回的错误信息，并停止本次粘贴。

## 测试方案

### 前端测试

- 右键文件显示统一一级菜单。
- 右键文件夹显示统一一级菜单。
- 复制、剪切后剪贴板状态正确更新。
- 有剪贴板时粘贴可用；无剪贴板时不可用或不触发。
- 文件右键粘贴到所在目录。
- 文件夹右键粘贴到该目录。
- 冲突时弹出包含源路径、目标路径、冲突数量的覆盖/副本/取消选择。
- 多冲突时支持全部覆盖、全部粘贴成副本和逐项处理。
- 剪切成功后清空剪贴板。
- 复制成功后保留剪贴板。
- 成功后刷新文件树。

### 后端测试

- 复制文件到空目录。
- 复制文件夹到空目录。
- 剪切文件到空目录。
- 剪切文件夹到空目录。
- 冲突预检返回完整冲突列表。
- 同名冲突返回可识别错误。
- 选择覆盖时可替换目标。
- 选择副本时生成不冲突的新名称。
- 拒绝目录粘贴到自身。
- 拒绝目录粘贴到子目录。
- 拒绝越界路径操作。

## 实现建议

推荐在前端新增一个局部“文件树剪贴板”状态，并把统一粘贴命令放到后端。

原因：

- 文件夹复制/剪切涉及递归和冲突处理，后端更适合做原子操作。
- 前端只需负责菜单、状态和冲突对话框，复杂度更可控。
- 统一后端命令可以减少未来扩展时的重复路径处理。

## 约束与注意事项

- 仅作用于辅助面板当前工作区文件树。
- 不影响目录浏览器、上传器、下载器和其它全局文件入口。
- 菜单文案与现有国际化体系保持一致。
- 新增函数需要补齐中文函数级注释，复杂逻辑需要补充行内说明。

## 已确认结论

本设计已确认：

- 复制、剪切、粘贴纳入文件操作菜单。
- 复制相对路径、复制绝对路径也统一放入同一一级菜单。
- 粘贴冲突时由用户选择全部覆盖、全部副本或逐项处理。
- 粘贴只作用于辅助面板当前工作区文件树。

## 后续增强：冲突预检与逐项处理

本增强已纳入当前计划：目录粘贴遇到多个同名项时，不再只能对顶层目标做一次覆盖/副本选择。后端必须提供预检能力，前端必须展示冲突数量和实际目标路径，并允许用户选择全部处理或逐项处理。
